# 硬盘可得容量计算方法

在购买或使用硬盘（HDD、SSD、移动硬盘、U 盘等）时，很多用户会发现：**实际可用容量往往比硬盘厂商标注的容量要小**。这并不是硬盘“偷工减料”，而是因为计算方法和文件系统的差异。本文将带你了解硬盘可得容量的计算方法，以及为什么会出现“容量缩水”的现象。

---

## 1. 硬盘容量的计算方式

### 1.1 厂商标注方式（十进制）

硬盘厂商一般采用 **十进制** 来计算容量：

1GB=1000×1000×1000 字节=109 字节1 \text{GB} = 1000 \times 1000 \times 1000 \ \text{字节} = 10^9 \ \text{字节}

因此，一个标注为 **1TB** 的硬盘，厂商的计算是：

1TB=1000 GB=1000×109 字节1 \text{TB} = 1000 \ \text{GB} = 1000 \times 10^9 \ \text{字节}

---

### 1.2 操作系统计算方式（二进制）

操作系统（如 Windows、Linux、macOS）通常使用 **二进制** 来计算容量：

1GiB=1024×1024×1024 字节=230 字节1 \text{GiB} = 1024 \times 1024 \times 1024 \ \text{字节} = 2^{30} \ \text{字节}

因此，操作系统会认为：

1GB=10243 字节≈1.074×109 字节1 \text{GB} = 1024^3 \ \text{字节} \approx 1.074 \times 10^9 \ \text{字节}

---

## 2. 实际可得容量计算

举个例子：
一个 **1TB（十进制）** 的硬盘：

- 厂商标注：

  1TB=1000×109 字节1 \text{TB} = 1000 \times 10^9 \ \text{字节}

- 操作系统计算：

  1000×10910243≈931 GiB\frac{1000 \times 10^9}{1024^3} \approx 931 \ \text{GiB}

所以在电脑中看到的可得容量大约是 **931GB**，而不是 1000GB。

---

## 3. 常见硬盘容量换算表

下表展示了常见硬盘标称容量与操作系统实际可见容量的对比（未考虑文件系统占用）：

| 标称容量（厂商标注） | 实际容量（操作系统显示，约） |
| -------------------- | ---------------------------- |
| 128GB                | 119GB                        |
| 256GB                | 238GB                        |
| 512GB                | 476GB                        |
| 1TB (1000GB)         | 931GB                        |
| 2TB (2000GB)         | 1862GB                       |
| 4TB (4000GB)         | 3725GB                       |
| 8TB (8000GB)         | 7451GB                       |

👉 你会发现，标称容量越大，**显示出来的差值也越大**。例如：1TB 少了约 69GB，4TB 少了约 275GB。

## 4. 文件系统占用

除了进制差异，硬盘在格式化时，还会被文件系统（NTFS、exFAT、ext4 等）占用一部分空间，例如：

- 文件分配表（FAT/NTFS 索引区）
- 日志区（ext4 的 journaling）
- 系统保留区

因此实际可得容量会再减少 **1% ~ 3%**。

---

## 5. 总结

硬盘“缩水”的原因主要有两点：

1. **进制差异**
   - 厂商：十进制（1GB = 1000³ 字节）
   - 系统：二进制（1GiB = 1024³ 字节）
2. **文件系统开销**
   - 格式化后，部分空间用于存储文件系统元数据。

👉 所以，一个标注 **1TB** 的硬盘，操作系统实际显示大约 **931GB** 是正常现象。

---

实际容量 (GiB)≈标称容量 (GB)×0.931\text{实际容量 (GiB)} \approx \text{标称容量 (GB)} \times 0.931
